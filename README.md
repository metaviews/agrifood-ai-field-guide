# Agrifood AI Field Guide

A knowledge base and presentation framework for public education on artificial intelligence across the agrifood sector.

**Start here:** [VISION.md](VISION.md) — the framing, audience, stance, taxonomy, freshness rules, and positioning relative to existing work.

## Layout

- `VISION.md` — the project's framing and operating principles
- `scans/` — environmental scan outputs (25 cycles through July 2026)
- `taxonomy/v5.md` — current taxonomy (fifth iteration). `taxonomy/v1.md..v4.md` retained as change-history anchors. Adds `maturity-scale`, `maturity-verification`, `maturity-longevity`, and `maturity-translation` cross-cutting tags for four-dimension maturity assessment.
- `units/` — content units (157 entries) — the substrate from which talks are built. Vendor deployments, NA/EU/LAC/Africa/MENA/Asia academic institutes, data cooperatives, IDSov frameworks, critical voices, ecosystem mappings, hyperscaler and EU regulatory substrate.
- `quotes/` — speaker voices (30 entries across 5 categories)
- `talks/` — methodology layer: six working archetypes + cluster-pattern taxonomy. See `talks/README.md` for operating principles.
- `GAPS.md` — corpus-wide gap registry (311 G-NNN IDs indexed with best-available descriptions)
- `LEADS.md` — corpus-wide lead registry (32 leads surfaced during cycle work, with realized/open status)
- `LICENSE` — CC BY 4.0 for content
- `NOTICE` — MIT for code

## Status

Substantively mature. Layer 1 (content library: units + scans + quotes) and Layer 2 (methodology + templates + archetypes) are well populated. Layer 3 (deck-generation tooling per VISION.md architecture) is documented but not built.

The corpus is structural: scans synthesize research, units anchor specific actor-networks or deployments, quotes capture speaker voices, talks compose units into presentations. `GAPS.md` and `LEADS.md` make the corpus's gap and lead tracking queryable.

## Contributing

Open issues, suggestions, and pointers to existing work welcome. See `VISION.md` for what this project is and isn't. When proposing new gaps or leads, follow the formatting conventions documented in `GAPS.md` and `LEADS.md` so the registries stay in sync.
